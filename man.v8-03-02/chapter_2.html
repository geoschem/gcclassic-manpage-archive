2. Installation

2.1 "Public" vs "beta" GEOS–Chem versions

When we have many updates which need to be added into the GEOS-Chem standard code, it is often expedient for us to split these among several "beta" GEOS-Chem versions. This allows us to test each internal version individually with the usual 1-month benchmark simulation. For some GEOS-Chem versions (especially those with updates that need to be tested over a longer time period), we will also perform a 1-year benchmark simulation. After iterating through several "beta" versions, when the code has attained stability, we will then announce a public GEOS–Chem version release with the customary fanfare. The GEOS–Chem User Guide (i.e. this document) is modified for each public version, but not for each "beta" version. To do so would probably lead to mass confusion among our user base.

Bob Yantosca and Claire Carouge will announce each GEOS–Chem beta version release via the GEOS–Chem website or email list. You may then obtain the "beta" version and begin using it. However, please note some aspects of each "beta" version may not be covered in the online documentation.

2.2 Obtaining the GEOS–Chem Source Code

From version 8-03-01, there are two possibilities for obtaining the GEOS–Chem source code and run directory:

  • use Git to clone the public repository for GEOS–Chem
  • download a tarball file over the network

We recommend users to use Git for obtaining the GEOS–Chem source code and run directory as well as maintaining their own local version of GEOS-Chem.

2.2.1 Dowloading a tarball file

The most recent GEOS–Chem release is available for download from the GEOS–Chem Source Code and Data Files page. You may also download data and input files for the GEOS–Chem model from this site.

The current GEOS–Chem source code and benchmark simulation output are contained in a GNU-zipped TAR file (i.e. the extension is *.tar.gz), which is also known as "tarball" format. To unzip the file, type: 
gunzip GEOS–Chem.v8–03–01-public.stdrun.tar.gz

and to extract the files, type:

tar xvf GEOS–Chem.v8-03–01-public.stdrun.tar

Downloading the tarball file will create a directory named v8–03–01 (or LOCAL_DIR_NAME), which is further divided into subdirectories:

Directory Description
Code.v8-03-01 Directory containing the GEOS–Chem source code
IDL Directory containing IDL code to process benchmark output
runs/v8-03-01 Contains the run directories for the various met field types
runs/v8-03-01/geos5 Contains output from v8–03–01 full-chemistry benchmark simulations done with  GEOS–5 met fields.

Each GEOS–Chem version is benchmarked with a standard full-chemistry simulation so that the results of a given version may be compared to the results of the prior version. In this way, errors in coding, emissions, or chemistry mechanisms may be readily discerned.

NOTE: The tarball method will be discontinued in the near future.

2.2.2 Clone the repository with Git

To clone the public GEOS–Chem source code repository, you simply have to type:

git clone git://git.as.harvard.edu/bmy/GEOS-Chem/ LOCAL-DIR-NAME

where LOCAL-DIR-NAME is the name you chose for your local directory to contain GEOS-Chem. Cloning the repository will simply create a directory (in fact a Git local repository) containing the last version of GEOS-Chem.

NOTE: For users located at Harvard, you need to access the public repository from behind the firewall. The path to the repository is different. Please refer to the GEOS-Chem wiki page for Git for more information.

2.3 Obtaining the GEOS–Chem run directories

Each of the GEOS-Chem run directories is saved as a separate Git repository, rather than putting them into a single repository. This is because some of the files (i.e. restart files) can be very large.

The download options are: 

  git clone git://git.as.harvard.edu/bmy/GEOS-Chem-rundirs/DIRECTORY-OPTION LOCAL-DIR-NAME

where DIRECTORY-OPTION may be one of the following:

2x2.5/geos4/SOA 2 x 2.5 GEOS-4 fullchem w/ SOA simulation
2x2.5/geos4/dicarbonyls 2 x 2.5 GEOS-4 fullchem w/ dicarbonyls option
2x2.5/geos4/isoprene 2 x 2.5 GEOS-4 fullchem w/ Caltech isoprene scheme
2x2.5/geos4/standard 2 x 2.5 GEOS-4 fullchem standard (43 tracers)

2x2.5/geos5/SOA 2 x 2.5 GEOS-5 fullchem w/ SOA simulation
2x2.5/geos5/dicarbonyls 2 x 2.5 GEOS-5 fullchem w/ dicarbonyls option
2x2.5/geos5/isoprene 2 x 2.5 GEOS-5 fullchem w/ Caltech isoprene scheme
2x2.5/geos5/standard 2 x 2.5 GEOS-5 fullchem standard (43 tracers)

4x5/geos4/SOA 4 x 5 GEOS-4 fullchem w/ SOA simulation
4x5/geos4/dicarbonyls 4 x 5 GEOS-4 fullchem w/ dicarbonyls option
4x5/geos4/isoprene 4 x 5 GEOS-4 fullchem w/ Caltech isoprene scheme
4x5/geos4/standard 4 x 5 GEOS-4 fullchem standard (43 tracers)

4x5/geos5/SOA 4 x 5 GEOS-5 fullchem w/ SOA simulation
4x5/geos5/dicarbonyls 4 x 5 GEOS-5 fullchem w/ dicarbonyls option
4x5/geos5/isoprene 4 x 5 GEOS-5 fullchem w/ Caltech isoprene scheme
4x5/geos5/standard 4 x 5 GEOS-5 fullchem standard (43 tracers)

benchmark 4 x 5 GEOS-5 1-month benchmark simulation

LOCAL-DIR-NAME is the name you chose for your local directory to contain GEOS-Chem.

GEOS–Chem restart files are provided in order to provide you with a set of initial conditions that you can start your model runs with. However, it is highly recommended that you not use these restart files for any of your production runs. Rather, you will probably want to generate your own restart file by spinning up the model for at least a year.

If you need a restart file for an offline simulation (e.g. offline aerosol, mercury, tagged CO, etc), you can use the GAMAP routine make_restart.pro to create a "fake" restart file. You can also ask the other members of the GEOS–Chem Mailing List if they have a restart file for the particular type of simulation that you are trying to do.

NOTE: If you are using GEOS–Chem with secondary aerosol tracers, then you will also need to create a soaprod.YYYYMMDD file. This can be done with GAMAP routines regridh_restart.pro, regridv_restart.pro and rewrite_agprod.pro or directly from GEOS-chem. Please see the GEOS–Chem wiki page for more information.

NOTE: For users located at Harvard, you need to access the public repository from behind the firewall. The path to the repository is different. Please refer to the GEOS-Chem wiki page for Git for more information.

2.4 Obtaining the GEOS–Chem data directories

You may now download all of the files from one of the GEOS–Chem data archives:

  1. Harvard archive (ftp.as.harvard.edu)
  2. Dalhousie archive (ucis.rain.dal.ca)

The Dalhousie archive is not as comprehensive as the Harvard archive. However, the Dalhousie archive stores data files for the various nested grids and global 1° x 1.25° grids that are not available on the Harvard archive.

We recommend using the Unix wget utility for downloading data. The appropriate command to download data from the Harvard data archive is:

wget -r -nH "ftp://ftp.as.harvard.edu/pub/geos-chem/data/DIRECTORY_NAME/"

And to download data from the Dalhousie data archive, use this command:

wget -r -nH "ftp://ucis.rain.dal.ca/DIRECTORY_NAME/"

Please see the following links for more information:

  1. Chapter 4: GEOS–Chem data directories
  2. GEOS–Chem wiki: Downloading GEOS–Chem data directories
  3. GEOS–Chem wiki: Using wget to download data
